# TL;DR

redis는 NoSql이며, Key-Value 형태의 데이터베이스이며, REmote DIctionary Server의 약자이다. 또한 Key로는 주로 String이 사용되지만, Value에는 다양한 타입이 올 수 있다.

## 특징

인터넷에 검색을 하다보니 다음과 같은 특징이 있다고 한다.

- 영속성을 지원하는 인메모리 데이터 저장소
- 읽기 성능 증대를 위한 서버 측 복제 지원
- 쓰기 성능 증대를 위한 클라이언트 측 sharding 지원

### 영속성?

redis에서 영속성은 지속성 보장을 위해 데이터를 디스크에 저장할 수 있음을 나타낸다. 서버가 다운되더라도 저장된 데이터를 불러올 수 있다.

### 복제?

redis의 복제는 이중화로 거의 실시간(비동기)으로 다른 레디스 노드에 복사하는 방식이다. 만약에 메인 노드가 중단되더라도 복제된 노드에서 서비스가 진행될 수 있다. 기본적으로 복제 서버는 읽기만 가능하다고 하는데 알아봐야 될 것 같다.

### 샤딩?

Partitioning이라고 부른다고도 하며, 데이터를 특정 조건에 따라 분산 저장하고, 상대적으로 더 많은 데이터를 저장할 수 있게 된다고 한다. 이러한 샤딩은 두 종류로 나뉜다고 한다.

1. 서버 측 샤딩
   여러 개로 구성된 클러스터에 데이터를 저장할 때 클러스터 레벨에서 데이터가 저장될 위치를 결정하는 것을 말한다.
2. 클라이언트 측 샤딩
   클라이언트 측 샤딩도 두 종류로 나뉜다고 한다...
   - 수직 샤딩
     - 관계형 데이터베이스의 테이블에 해당하는 정보를 노드별로 분할하는 방법이다.
   - 범위 지정 샤딩
     - 키를 특정 범위를 기준으로 분할하여 저장하는 방법이다.
   - 해시 기반 샤딩
     - 키를 해시 함수를 이용하여 시스템만의 특정 연산을 추가해서 데이터의 위치를 결정하는 방법이다.
     - 어떤 블로그에서 해당 예시를 들었다. ex. 샤드 번호 = 해시함수("값") mod(%) 10
